# Tech Stack

We need to understand the project's technology choices to provide relevant guidance.

## What to Clarify

- **Languages** - What programming languages does this project use?
- **Primary language** - Which language is the main one?
- **Frameworks** - What frameworks or libraries are used?
- **Package manager** - How are dependencies managed? (npm, yarn, pip, cargo, etc.)
- **Build tool** - How is the project built? (webpack, vite, make, etc.)

## Optional

- **Test framework** - What testing tools are used?
- **Linting/formatting** - Code quality tools? (eslint, prettier, black, etc.)
- **Database** - What database(s) if any?
- **Runtime** - Node version, Python version, etc.?
- **Monorepo** - Is this a monorepo? What tool manages it?

## Examples

**Good languages answer:**
> TypeScript for the frontend and backend, Python for ML pipelines

**Good frameworks answer:**
> React with Next.js for frontend, FastAPI for backend API, PyTorch for ML

**Good package manager answer:**
> pnpm for the monorepo, poetry for Python packages

**Good test framework answer:**
> Jest for unit tests, Playwright for E2E, pytest for Python
